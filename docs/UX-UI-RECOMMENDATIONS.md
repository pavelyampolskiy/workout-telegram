# UX/UI рекомендации — Workout Telegram Mini App

Обзор выполнен с точки зрения UX/UI дизайнера по текущей кодовой базе и паттернам приложения.

---

## Что уже хорошо

- **Единые токены**: `design-tokens.js` и `shared` задают цвет, карточки, типографику — база для консистентности.
- **Safe areas**: учёт `safe-top`, `safe-bottom`, `safe-top-lg` под Telegram и вырезы.
- **Тактильный фидбек**: `card-press` (scale + opacity), местами HapticFeedback — понятная реакция на тап.
- **Микроанимации**: counter-pop, save-flash, row-in, toast, modal — делают интерфейс живым.
- **Тёмная тема**: единый тёмный стиль, читаемость на фоне фото зала.

---

## 1. Консистентность

| Проблема | Рекомендация |
|----------|--------------|
| Разный верхний отступ: часть экранов `safe-top`, часть `safe-top-lg` | Зафиксировать один стандарт: например, все списковые/контентные экраны — `safe-top-lg`, заголовок с `pt-6`. |
| Подписи секций (Unlocked, In Progress, By type): то есть, то убраны | Либо везде короткие подписи над блоками (как сейчас в Achievements), либо убрать там, где дублируют табы/заголовок. |
| Иконки: часть через компоненты, часть inline SVG | Вынести повторяющиеся иконки (день, календарь, стрелки) в `Icons.jsx` и использовать везде оттуда. |

---

## 2. Иерархия и читаемость

| Проблема | Рекомендация |
|----------|--------------|
| Много приглушённого текста (`text-white/25`, `text-white/40`) на главном экране | Оставить «ARE YOU» и статистику приглушёнными, но основные действия (New Workout, 4 кнопки) — не ниже `text-white/50`, иначе теряется приоритет. |
| На экране упражнения убрана кнопка «Назад» | Пользователь может не знать, что выход — через «Finish Exercise». Добавить короткую подсказку («Swipe back» или иконка «завершить» в шапке) или снова лёгкий «назад» (только стрелка). |
| Длинные названия упражнений/дней | Задать `line-clamp` (1–2 строки) и `truncate` для карточек, чтобы не ломать сетку. |

---

## 3. Навигация и ориентация

| Проблема | Рекомендация |
|----------|--------------|
| Нет явного «хлебного следа» на вложенных экранах (Day → Exercise, Program → Day) | На экранах 2–3 уровня добавить компактную шапку: «Day A › Leg Press» или «Program › Day B» — без лишней кнопки «Назад», но с контекстом. |
| History / Statistics: после выбора месяца/периода неочевидно, что это фильтр | Добавить лёгкий индикатор активного фильтра (например, точку или подчёркивание под «March 2026» / «Week») или короткий текст «Showing: March 2026». |
| Recovery Check перед каждой тренировкой может раздражать | Дать возможность «Skip recovery check today» один раз или запоминать «Don’t ask again today» после первого скипа. |

---

## 4. Обратная связь и состояния

| Проблема | Рекомендация |
|----------|--------------|
| Сохранение сета: только save-flash, без явного «Saved» | Краткий toast «Set saved» (1–2 сек) или галочка рядом с последним сетом. |
| Загрузка (skeleton) без прогресса на долгих запросах | На экранах с тяжёлыми запросами (History с фильтром, Program) — спиннер или «Loading…» под табами, чтобы было понятно, что идёт обновление. |
| Кнопки «Save» / «Finish» при отправке | Всегда показывать loading (disable + спиннер или «…»), чтобы не было двойного тапа. |
| Пустые списки (нет тренировок, нет упражнений в дне) | Уже есть сообщения — убедиться, что везде есть CTA: «Start first workout», «Add first exercise». |

---

## 5. Доступность (a11y)

| Проблема | Рекомендация |
|----------|--------------|
| Модалки: focus trap есть в ConfirmModal, не везде | Во всех модальных окнах (Add Day, Rename, Delete, Recovery) использовать `useFocusTrap` и закрытие по Escape. |
| Кнопки только с иконкой (карандаш, корзина, «+») | Добавить `aria-label` («Rename day», «Delete day», «Add exercise») для скринридеров и подсказок. |
| Интерактивные карточки (Day A, History row) | Либо `role="button"` + `tabIndex={0}` + `onKeyDown` (Enter/Space), либо оставить только клик, но тогда не вешать семантику button на div. Лучше явные `<button>` или `role="button"` + доступ с клавиатуры. |
| Контраст текста | Проверить комбинации `text-white/25` и `text-white/30` на тёмном фоне (WCAG AA): при необходимости поднять до 0.4 для важного вторичного текста. |

---

## 6. Пустые и ошибочные состояния

| Проблема | Рекомендация |
|----------|--------------|
| ErrorScreen: единый вид, но текст «Something went wrong» общий | Для типичных ошибок (сеть, 401) показывать короткое сообщение + «Retry»; для остальных — общий текст + при желании «Report» (например, ссылка в Telegram). |
| Пустой Program (нет дней) | Вместо только «Add day» показать один раз блок: «No days yet. Add your first day (e.g. Day A) and add exercises.» + кнопка «Add day». |
| History пустой за месяц | Текст «No workouts yet» ок; можно добавить «Workouts you finish will appear here.» |

---

## 7. Восприятие производительности

| Проблема | Рекомендация |
|----------|--------------|
| Переход на экран упражнения после выбора дня | Пока грузится программа, показывать скелетон списка упражнений (как DaySkeleton), а не пустой экран. |
| Переключение табов (History, Statistics) | Уже есть скелетоны — при смене таба не сбрасывать контент в пустой вид: оставить предыдущие данные до загрузки новых (или мелкий скелетон поверх). |

---

## 8. Специфика Telegram Mini App

| Проблема | Рекомендация |
|----------|--------------|
| Кнопка «Close» в шапке Telegram | Не дублировать свою «Back» на главном экране; на вложенных — либо полагаться на жест/системную кнопку, либо один минимальный «‹» при необходимости. |
| Расширение окна (expand()) | Вызывать при первом входе или после первого завершённого сета, чтобы дать больше места контенту. |
| Цвет заголовка/фона под бренд бота | Использовать `ThemeParams` (если нужно) для акцентного цвета, чтобы Mini App не выглядел чужеродно в чате. |

---

## Приоритеты внедрения

1. **Высокий**: aria-labels на иконках, контраст важного текста, явная обратная связь при сохранении сета, один стандарт safe-top.
2. **Средний**: контекст в шапке на вложенных экранах (Day › Exercise), улучшение пустых состояний с CTA, focus trap и Escape во всех модалках.
3. **Низкий**: хлебные крошки, расширение окна Telegram, тонкая настройка скипа Recovery Check.

Документ можно дополнять по мере изменений в приложении.
